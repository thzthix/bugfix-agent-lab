# bugfix-agent-lab

`Responses API` 기반의 버그 수정 에이전트 하네스를 설계하고 구현하는 실습 저장소입니다.

이 프로젝트의 목표는 에이전트가 테스트 실패를 바탕으로 원인을 추론하고, 제한된 범위 안에서 코드를 수정한 뒤, 테스트를 다시 실행하면서 스스로 수정해 나가는 루프를 만드는 것입니다. 최종적으로 에이전트는 구조화된 결과 JSON과 사람이 읽을 수 있는 실행 기록 Markdown을 남기도록 설계합니다.

## 실습 목표

- `Responses API` 기반의 멀티턴 에이전트 루프 설계
- 제한된 도구 집합을 사용하는 코드 수정 워크플로 구성
- 테스트 결과를 기준으로 한 자기 수정 루프 구현
- 구조화된 결과물과 실행 보고서 산출
- 에이전트 권한 범위를 명시적으로 제한하는 하네스 설계

## 아키텍처 개요

이 저장소는 하네스 코드와 수정 대상 프로젝트를 분리합니다.

- `harness/`
  에이전트 루프와 도구 실행을 담당하는 코드
- `target-project/`
  버그가 있는 연습용 Python 프로젝트
- `outputs/`
  실행 결과와 보고서 산출물 저장 위치

이 구조를 통해 에이전트가 무엇을 읽고, 무엇을 수정할 수 있는지 명확하게 통제할 수 있습니다.

## 디렉터리 구조

```text
bugfix-agent-lab/
  harness/
  target-project/
    docs/
    src/
    tests/
  outputs/
  README.md
```

## 실행 산출물

이 실습은 성공 또는 실패 후 다음 산출물을 남기는 것을 목표로 합니다.

- `outputs/result.json`
- `outputs/report.md`

`result.json`은 구조화된 실행 결과를 담고, `report.md`는 실패 원인, 시도한 방법, 대안, 선택 이유, 실제 구현 내용을 사람이 읽을 수 있게 정리한 문서입니다.

## 진행 방식

이 저장소는 단계별 브랜치 방식으로 진행합니다.

원칙:
- 작업 단계마다 별도 브랜치를 생성한다.
- 설계를 먼저 확정한다.
- 수도 코드로 흐름을 점검한다.
- 실제 구현은 그 다음 단계에서 진행한다.

## 다음 단계

초기 구조를 만든 뒤 다음 순서로 진행합니다.

1. 버그가 있는 연습용 Python 프로젝트 추가
2. 테스트 스크립트와 검증 기준 확정
3. 하네스 도구 정의
4. 자기 수정 루프 구현
5. 결과 JSON 및 보고서 생성
